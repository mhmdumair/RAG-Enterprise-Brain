# Project Summary: RAG Enterprise Brain

This file provides a file-based explanation of every file in the project. The project is a Retrieval-Augmented Generation (RAG) system for processing and querying PDF documents using vector embeddings and extractive QA.

## Root Directory Files

- **.env**: Environment variables file for configuration (e.g., database URIs, API keys).
- **.env.example**: Template for environment variables, showing required settings without sensitive data.
- **.gitignore**: Specifies files and directories to ignore in Git version control (e.g., .env, __pycache__, node_modules).
- **README.md**: Project documentation (currently empty).
- **ask.py**: Standalone script for querying the system, likely for testing or CLI usage.
- **find_chunk.py**: Script to find specific chunks in the vector index or database.
- **pytest.ini**: Configuration file for pytest testing framework.
- **requirements-dev.txt**: Python dependencies for development (e.g., testing tools).
- **requirements.txt**: Python dependencies for production (e.g., FastAPI, PyMuPDF, transformers).
- **tasks.md**: Detailed task breakdown and development phases for the project.
- **test_vector_retrieval.py**: Test script for vector retrieval functionality.

## api/ Directory (FastAPI Backend)

- **__init__.py**: Marks the directory as a Python package.
- **dependencies.py**: FastAPI dependency injection functions (e.g., database client, model instances).
- **main.py**: Main FastAPI application file, wires routes, middleware, and lifespan events.
- **middleware.py**: FastAPI middleware for CORS, request timing, and error handling.
- **schemas.py**: Pydantic models defining request/response schemas for API endpoints.
- **routes/__init__.py**: Marks routes as a package.
- **routes/debug.py**: Debug-related API routes (e.g., for testing or introspection).
- **routes/documents.py**: API routes for document management (e.g., list ingested PDFs).
- **routes/health.py**: Health check endpoint to verify API, DB, and models are operational.
- **routes/ingest.py**: API route for ingesting PDF uploads and triggering the processing pipeline.
- **routes/query.py**: API route for querying the system and returning verified answers.

## auditor/ Directory (QA and Validation Layer)

- **__init__.py**: Marks the directory as a Python package.
- **abstention.py**: Implements abstention logic to accept or reject answer spans based on confidence scores.
- **deduplicator.py**: Removes duplicate answer spans using SHA-256 hashing.
- **dispatcher.py**: Orchestrates parallel QA workers to process chunks concurrently.
- **qa_model.py**: Loads and runs the extractive QA model (roberta-base-squad2) for answer extraction.
- **rake_fallback.py**: Fallback mechanism using RAKE keyword extraction to reformulate queries.
- **retriever.py**: Performs approximate nearest neighbor search in the FAISS index to retrieve relevant chunks.
- **worker.py**: Asynchronous worker for running QA on individual chunks.

## brain/ Directory (Ingestion and Indexing)

- **__init__.py**: Marks the directory as a Python package.
- **chunker.py**: Splits parsed PDF pages into overlapping text chunks using sliding window technique.
- **embedder.py**: Converts text chunks into vector embeddings using MiniLM-L6-v2 model.
- **indexer.py**: Builds and persists the FAISS vector index for efficient similarity search.
- **parser.py**: Parses PDF files using PyMuPDF to extract text blocks and bounding box coordinates.
- **pipeline.py**: Orchestrates the full ingestion pipeline: parse → chunk → embed → index → store.
- **store.py**: Stores chunk metadata in MongoDB, mapping vector IDs to filenames, pages, and BBoxes.

## core/ Directory (Shared Utilities)

- **__init__.py**: Marks the directory as a Python package.
- **config.py**: Central configuration management for settings, environment variables, and thresholds.
- **exceptions.py**: Custom exception classes for domain-specific errors (e.g., IngestionError, QAError).
- **logger.py**: Structured logging setup used across all modules.
- **utils.py**: Utility functions for text cleaning, hashing, and bounding box normalization.

## db/ Directory (Database Layer)

- **__init__.py**: Marks the directory as a Python package.
- **client.py**: Asynchronous MongoDB client connection using Motor (singleton pattern).
- **models.py**: Defines MongoDB collection schemas for documents, chunks, and results.
- **queries.py**: Reusable database query functions (e.g., insert chunk, find by vector ID).
